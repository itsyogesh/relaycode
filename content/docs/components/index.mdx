# Component Overview

Relaycode ships a library of **22 input components** and **10 contextual selectors** that map directly to Substrate runtime types. Each component understands the SCALE type it represents, validates input accordingly, and emits values ready for SCALE encoding.

Components are resolved automatically through a [priority-based type resolution system](/docs/api/input-map#type-resolution).

## Available Components

| Component | Substrate Types | Description |
|---|---|---|
| [AccountInput](/docs/components/account-input) | `AccountId`, `MultiAddress`, `Address` | Address input with wallet integration and SS58 validation |
| [BalanceInput](/docs/components/balance-input) | `Balance`, `BalanceOf`, `Compact<Balance>` | Token amount with denomination selector and balance display |
| [AmountInput](/docs/components/amount-input) | `u8`-`u128`, `i8`-`i128`, `Compact<uN>` | Numeric integer input for unsigned and signed integers |
| [BoolInput](/docs/components/bool-input) | `bool` | Toggle switch for boolean parameters |
| [HashInput](/docs/components/hash-input) | `H160`, `H256`, `H512`, `Hash` | Fixed-length hex hash input with length validation |
| [BytesInput](/docs/components/bytes-input) | `Bytes`, `Vec<u8>` | Raw bytes hex input with multiple input modes |
| [CallInput](/docs/components/call-input) | `Call`, `RuntimeCall` | Nested extrinsic builder for composing calls |
| [EnumInput](/docs/components/enum-input) | Enum types from metadata | Variant selector with dynamic inner value resolution |
| [OptionInput](/docs/components/option-input) | `Option<T>` | Optional value toggle wrapping any inner type |
| [StructInput](/docs/components/struct-input) | Struct types from metadata | Named field grouping with recursive sub-inputs |
| [TextInput](/docs/components/text-input) | `String`, `Text` | Basic string input and fallback for unknown types |
| [TupleInput](/docs/components/tuple-input) | `(T1, T2, ...)` | Positional element inputs for tuple types |
| [VectorInput](/docs/components/vector-input) | `Vec<T>`, `BoundedVec<T>`, `BTreeSet<T>` | Dynamic-length array with add/remove controls (supports unique mode for sets) |
| [VectorFixedInput](/docs/components/vector-fixed-input) | `[T; N]` | Fixed-length array with N input slots |
| [BTreeMapInput](/docs/components/btree-map-input) | `BTreeMap<K, V>` | Key-value pair entries |
| [MomentInput](/docs/components/moment-input) | `Moment` | Timestamp input with date picker and presets |
| [VoteInput](/docs/components/vote-input) | `AccountVote` | Voting preference input (Standard/Split) |
| [VoteThresholdInput](/docs/components/vote-threshold-input) | `VoteThreshold` | Threshold selector (SuperMajorityApprove/Against/SimpleMajority) |
| [KeyValueInput](/docs/components/key-value-input) | `KeyValue` | Single key-value pair input |

For the full priority ordering and pattern matching details, see the [Input Map API reference](/docs/api/input-map#priority-order).

## Shared Props

Every input component implements the `ParamInputProps` interface:

```ts
interface ParamInputProps {
  name: string;
  label?: string;
  description?: string;
  typeName?: string;
  isDisabled?: boolean;
  isRequired?: boolean;
  error?: string;
  client: DedotClient<PolkadotApi>;
  typeId?: number;
  value?: any;
  onChange?: (value: unknown) => void;
  palletContext?: PalletContextData | null;
  isContextLoading?: boolean;
}
```

See individual component pages for type-specific props and behavior details.

## Contextual Selectors

When the extrinsic builder detects specific pallet and method combinations, it replaces generic inputs with **contextual selectors** that display live chain data. These are resolved via `findComponentWithContext` before falling back to the standard registry.

| Selector | Pallets | Description |
|---|---|---|
| [ValidatorSelector](/docs/components/validator-selector) | Staking, NominationPools | Searchable combobox for validators (single-select and multi-select modes) |
| [PoolSelector](/docs/components/pool-selector) | NominationPools | Searchable dropdown for nomination pool selection |
| [ReferendumSelector](/docs/components/referendum-selector) | ConvictionVoting, Referenda | Dropdown for selecting active referenda |
| [BountySelector](/docs/components/bounty-selector) | Bounties | Selector for on-chain bounty IDs |
| [TrackSelector](/docs/components/track-selector) | ConvictionVoting | Dropdown for OpenGov track selection |
| [EraSelector](/docs/components/era-selector) | Staking | Era index input with current era context |
| [ContextHint](/docs/components/context-hint) | Various | Informational display for contextual data |
| [SelectorFallback](/docs/components/selector-fallback) | Various | Loading/error state wrapper for selectors |
